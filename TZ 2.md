# Новые сущности

1 Locker — Шкафчик

| Поле        | Тип     | Описание | 
|-------------|---------|-------------|
| id          | UUID    | Уникальный идентификатор|
| number     | Integer  | Номер шкафчика (1–20, уникален)           |
| client_id        | UUID  | Связь с клиентом (один-к-одному, NULL если свободен)           | 

2 Service — Дополнительная услуга

| Поле        | Тип     | Описание | 
|-------------|---------|-------------|
| id  | String  | Уникальный идентификатор (латинские символы, напр. POOL)           | 
| name    | String    | Название услуги на русском           |
| price    | Integer    | Цена за посещение в рублях           |

3 Client_Service — Связь клиент–услуга

| Поле        | Тип     | Описание | 
|-------------|---------|-------------|
| client_id       | UUID  | Идентификатор клиента           | 
| service_id       | String  |   Идентификатор услуги         | 

4 Seed-данные для Service

| ID услуги        | 	Название	    | Цена, руб. | 
|-------------|---------|-------------|
|  SOLARIUM  | Солярий | 400 | 
| POOL  | Бассейн    | 200           |
| SAUNA       | Сауна  |   0 (бесплатно)         | 
| CRYOSAUNA       | Криосауна  |   1000        | 
| CROSSFIT       | Кроссфит  |   500         | 

# Итоговая карта связей

| Связь | Тип                                        |
|-----|-------------------------------------------------|
| Тренер → Клиент | Один ко многим                  |
| Клиент → Шкафчик | Один к одному                       |
| Клиент ↔ Услуга | Многие ко многим (через Client_Service)              |

# Новые и изменённые эндпоинты

были добавлены или изменены следующие эндпоинты

| Метод       | 	Путь	    | Описание | 
|-------------|---------|-------------|
|  POST  | /api/clients/{clientId}/locker/{lockerId} | Назначить шкафчик клиенту | 
| POST  | /api/clients/{clientId}/additionalServices/{serviceId}    | Добавить услугу клиенту           |

Ответом будет Ответ GET /api/clients/{id}/detail теперь должен включать также шкафчик и список услуг:
{ ..., "trainer": {...}, "locker": { "id": "...", "number": 5 }, "services": [{ "id": "POOL", "name": "Бассейн", "price": 200 }] }

5 Шкафчики 

| Поле        | Тип     | Описание | 
|-------------|---------|-------------|
| GET       | /api/lockers  | Список всех шкафчиков со статусом (свободен / занят клиентом)           |

6 Услуги 

| Метод       | 		Путь		  | Описание | 
|-------------|---------|-------------|
|  GET  | /api/additionalServices | Список всех услуг с ценами | 
| GET  | /api/additionalServices/{id}    | Информация об услуге + список подписанных клиентов           |

# Бизнес-правила для новых операций
•	Назначить шкафчик клиенту можно только если шкафчик свободен (client_id = null) → иначе HTTP 409.
•	Один клиент не может иметь более одного шкафчика → при повторном назначении освобождать предыдущий или возвращать ошибку (на выбор студента).
•	Добавить уже подключённую услугу повторно → HTTP 409 или игнорировать (на выбор).
